    InfoSec Best Practices

    Best practices for information security using reelyActive open source software.

    The TL;DR (Too Long; Didn't Read)

    This is a top 9 checklist we at reelyActive use to harden our system when we deploy our software.

    We recommend doing more hardening of your system, but those 9 recommendations are a good start.

    What's the expected result?
    The system will have less attack vectors and a better incidence respond capability.
    What's the alternative?
    Answer what they might be looking for instead.
    Is this for me?
    It is for anyone who do not have knowledge in security or just a little. It is an overview of the concept with examples. More detail will be added with time and if asked.

    Passwords   check 1 of 9

    In this section, we are going to talk about passwords. Things to think of when installing a new operating system or application.

    Why are password important?
    Because password is the main protector of our private information in life. Every account and system has a password, so knowing how to create great password and how to deal with them is important.

    Information Part 1

    Every year many security breaches are made because of bad password or poor passwords management. This is possible because people have the bad habit to reuse password, to write them down on paper or on a text document or using poor password policy. The help the following parts will guide you to create and manage password.

    Change all the default password!! Part 2

    Did you know that a lot of home router or selfhosted service still have default passwords? Even in enterprise network employee forgets to change passwords.

    If there is one thing to retain from this checklist it Change the default password. People tend to forget, and this is bad because it means that anyone with 5 minutes of research on a research engine (DuckDuckGo, Google, Ecosia) can log in your system or application.

    Password policy Part 3

    The first tab is what I recommend. The second tab is the minimum require.

    What is password policy? It is an arbitrary set of rules to create a password. They are put in place to make your password harder to guest or brute force.

    • Must be at least 15 characters or more.
    • Must contain at least 3 uppercase characters or more.
    • Must contain at least 3 numbers or more.
    • Must contain at least 3 symbols or more.
    • Should not use dictionary words in it.
    • Should be change every 30 days.

    An example of what could look the password:

     #N1xAV1!kjb^1@vG7d1$

    To crack a password like this would take a few quintillion. The time is relative to the common technology, but supercomputer exist, and quantum computer are getting faster.

    • Must be at least 8 characters or more.
    • Must contain at least 1 uppercase characters or more.
    • Must contain at least 1 numbers or more.
    • Must contain at least 1 symbols or more.
    • Should not use dictionary words in it.
    • Should be change every 90 days.

    An example of what could look the password:

     gUaf&3gp

    To crack a password like this would take a few hundred years. The time is relative to the common technology, but supercomputer exist, and quantum computer are getting faster.

    The more complex your password is the best it is. Changing password every 30 or 90 days take time and not a lot of people are doing it, but it is a good practice on important account.

    Password Manager Part 4

    I recommend using a password manager to store your passwords. I personally prefer open-source solution and selfhosted, but I know that not everyone have the knowledge or resources for it. So here is my top 3 of open-source and top 3 of proprietary software. (Not in order)

    1. Bitwarden
    2. KeyPassXC
    3. Psono
    1. LastPass
    2. 1Password
    3. DashLane

    Update and Upgrade   check 2 of 9

    In this section, we are going to talk about updates and upgrades.

    Why are update and upgrade important?
    Doing the update and upgrade is important because it will allow you to have the latest version and patch of the operating system and their package.

    Information Part 1

    Because of the lack of update and upgrade in many networks, attackers can get in. This is because older version does not have the latest security patch or bug fixes which means hackers can use many known vulnerabilities. If you update and upgrade everything regularly you considerably reduce the attack vector because they need to find new vulnerabilities.

    How to update and upgrade. Part 2

    If we are missing an operating system, let us know.

    Here are commands to update and upgrade your Linux operation system. You should check to new update and upgrade very regularly like everyday or week. 

     sudo apt update
     sudo apt upgrade
     
     sudo apk update
     sudo apk upgrade
     
     sudo pacman -Syy
     sudo pacman -Su
     
     sudo pkg update
     sudo pkg upgrade

    Centos and RHEL base until version 7 and Fedora until version 21: 

     sudo yum update
     sudo yum upgrade

    Centos and RHEL base version 8 and up and Fedora version 22 and up: 

     sudo dnf check-update
     sudo dnf upgrade

    You could automate those actions. Some package manager has their own automation, or you can use cron job.

    Update and Upgrade in a production environment Part 3

    Unfortunately, you cannot blindly update and upgrade in a production environment. You must be careful because sometimes it can break your application or operating system. To prevent this, you should test in your test or dev environment to make sure everything run, and you can read the note of the update and upgrade to see what the change are.

    Firewall   check 3 of 9

    In this section, we are going to talk about firewall.

    Why are firewall important?
    Firewalls are important because they help reducing the attack vector from all ports (65535) to only the public ports we chose to expose.

    Information Part 1

    A firewall is a network device use to analyse network traffic. It will not look in the packets data but only the source and destination IP address and port. Depending on the rules put in place, it will let the packet pass through or not. This is useful to prevent people to connect to other ports than suppose or to only let some IP address connect to the device.

    Use of firewall Part 2

    This is an example of firewall that can be use. (Do not exist for FreeBSD.)

    To demonstrate some rules for a firewall, we are going to use a simple Firewall. Uncomplicated Firewall (UFW) is a basic but well-known Firewall. We will show how to install and put in place simple rules that can be use on a device with Pareto-anywhere.

    sudo apt install ufw

    sudo ufw allow 50001/udp

    Backups   Check 4 of 9

    In this section, we are going to talk about backups.

    Why are backups important?
    They are a copy of your information in case of lost or attack. It will help you get back to work without losing everything.

    Information Part 1

    Backups are important not to protect your device before or during an attack but after. When an attacker can get in your systems he can erase or modify or encrypt your data. PenTest magazine said that in 2020 around 50% of the company add a ransomware attack in 2020 and the average ransom is 178 000$. Having backups taking regularly can save you from this. It is even useful if your device break or you lose all your data because of a wrong command.

    • Backups should be done at least once a day when possible.
    • Should have multiple copy of the backup on different site.
    • Keep a backup not connected to any network so it is isolated.

    Secure Shell Protocol   check 5 of 9

    In this section, we are going to talk about Secure Shell Protocol (ssh). Things to do of when installing a new operating system with ssh.

    Why do we need ssh?
    We use ssh to connect to a remote host. SSH is probably the most used services to configure and manage device through the network. It creates an encrypted connection between two systems and give us the possibility to execute commands remotely.

    Information Part 1

    Being a really used services a lot of attacker will try to connect on port 22 (default port of ssh) to get access to the host. To prevent and make their life hard we can change the default configuration to protect our service.

    Change some of the default configuration Part 2

    To modification that we recommend are the minimum recommended. There is a lot more configuration possible. To see other option, look here.

    ## Open /etc/ssh/sshd_config ## with your favorite text editor (nano, vim, ...)

    ## Make sure the Protocole 2 is enable and not Protocole 1
    ## Add the following ligne to the top of the if not found

    Protocol 2

    ## Change Default Port
    ##Replace the default "#Port 22" with any above 1024:

    Port 2445

    ## Don't allow Empty password connection
    ## Uncomment the following linge and put no if there is something else

    PermitEmptyPasssword no

    ## Don't allow connecting with Password
    ## Uncomment the following linge and put no if there is something else

    PasswordAuthentification no

    ## Don't allow X11 forwarding
    ## Uncomment the following linge and put no if there is something else

    X11Forwarding no

    ## Don't allow root to ssh
    ## Uncomment the following linge and put no if there is something else

    PermitRootLogin no

    ## Allow specific user to use ssh
    ## Replace <USERNAME> with the real username.

    AllowUsers <USERNAME> <USERNAME>

    ## Deny specific user to use ssh
    ## Replace <USERNAME> with the real username.

    DenyUsers <USERNAME> <USERNAME>

    ## Change log level (more verbose than default)
    ## Uncomment the following linge and put VERBOSE if there is something else

    LogLevel VERBOSE

    We disable the option to connect with ssh with our password. To be able to connect remotely we need to create a pair of keys. To do it go look at this tutorial.

    Logging   check 6 of 9

    In this section, we are going to talk about logs.

    Why are logs important?
    Loggins your system and services are important because it help knowing what is going on. With logs, you can see who is trying to connect to your device or see why did something crash.

    Information Part 1

    In most Linux distribution three services are used to log the kernel and user space. They are Syslog, Rsyslog and Journald. You can research how they work and how to set them up to get more information. Those information are useful to know what have done the attacker or why something is not work (ex: if it has crash). You can event you it to prevent some types of attack if you use a monitoring application (Logcheck, Logwatch, Logstash).

    Logs can be founds in /var/log folder of the operating system.

    Do not forget to enable logging in application that do not do it automatically.

    Users and Groups   check 7 of 9

    This part is more advance and require a more deeply understanding of linux and the operating system.

    In this section, we are going to talk about the possibility in managing users and groups of users. What should a use be able to access and what privilege should they have.

    Why is user and groups permission important?
    Controlling who can access what in a network or on a system is incredibly important because a person who has access to services, they do not need is a security risk and they could break things. Imagine a salesman having access to a production environment and enter a wrong command by accident.

    Information Part 1

    In a GNU / Linux system, there is three type of users: the root, the sudoers and normal users. The root user is the default all access user. The root user is the admin of the system and have “all the power”. The sudoers are users that in the sudo groups which give them the same privilege than the root. The normal users do not have access to privilege require services and only what they created or shared with them. The groups are like a tag added to users to set a set of rules to multiple people at the same time and give them access to files and folders.

    We recommend creating a user per person using the system and not a user for multiple people because you can not manage the access efficiently. And with this in mind block multiple connection from the same account on a single host.

    A default guideline would be:

    • One user per person
    • One connection per user
    • Users should not have all the root privilege with sudo command (except admins)
    • Make multiple small groups instead of one group with a lot of permissions.

    Making those changes is more advance but help a lot if an attacker gain access to the system because he will be limited in is action. To help you make these changes, here a link to the limits.conf man page (manual) and here a link to a tutorial on user and groups privilege management from Digital Ocean.

    Legacy package and services   check 8 of 9

    In this section, we are going to talk about why you should remove services and packege not used.

    Why is removing unsused packege and services important?
    Removing old things from a host is useful to reduce the attack vector and reduce the change of broken the environment.

    Information Part 1

    You should never have unused package and services on your system because it takes resources, have change to break the system with a new update or can contain vulnerabilities that can help the attacker get in. In many cases, when you install a new operation system it comes with default package like curl, wget, python and others. Those packages might not be useful on your system and having them give tool to an attacker.

    sudo apt remove <PACKAGE_NAME>
    sudo apk del <PACKAGE_NAME>
    sudo pacman -R <PACKAGE_NAME>

    Centos and RHEL base until version 7 and Fedora until version 21:

    sudo yum remove <PACKAGE_NAME>

    Centos and RHEL base version 8 and up and Fedora version 22 and up:

    sudo dnf remove <PACKAGE_NAME>

    Data Transmition   check 9 of 9

    In this section, we are going to talk about why you should always have encrypted communication.

    Why is encrypted communication important?
    Having none encrypted data is letting everyone know what we are sharing. This can be harmful because people can know classify information or even change the information transmitted.

    Information Part 1

    Sending clear text to over internet today is an invitation for attacker and a big lack of privacy. Transmitting information over internet should use ssl or tls (encryption method (https)) or a vpn. There is many other protocols to encrypt data but they are wildly use. You can use ssl / tls on a web server with the help of a certificate, it can be free and relatively easy to install. In the other end, using a vpn is mainly known today. Setting up a vpn in quite cheap and easy. These will protect your privacy and data from attacker or even anyone. In a corporate network, data does not have to be encrypted because it is a close network but if it is possible, it should be. Remember: Clear information is information that can be modify and leverage.

    • To get your basic certificate for ssl / tls, we would recommend Let’s encrypt.
    • To setup a vpn, we would recommend WireGuard or OpenVPN.
